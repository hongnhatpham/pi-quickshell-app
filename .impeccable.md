# Design Context

## Target audience
Personal use only. This app is for the user’s daily work with ARIA-03, Pi, delegated workers, and coding tasks on this machine.

## Use cases
- chat with Pi in a proper desktop app window instead of only the terminal
- observe spawned subagents and delegated task trees live
- inspect code diffs directly in the conversation context
- inspect loaded extensions and runtime/session settings
- keep conversation and orchestration in one coherent workspace

## Brand personality / tone
The interface should match the existing desktop environment already established by:
- Niri
- the current QuickShell bar
- Alacritty

Desired feeling:
- warm industrial
- sharp geometry
- compact and refined
- terminal-adjacent rather than glossy app-like
- calm, grounded, and capable
- no generic AI-chat product styling

## Visual constraints
- Use the existing desktop palette as the source of truth
- Keep geometry tight; no oversized rounded pills
- Use restrained orange accent energy rather than loud neon emphasis
- Avoid generic dashboard cards and trendy gradient-heavy AI aesthetics
- Typography should feel aligned with the current IBM Plex Mono-based shell language

## Layout reference
Reference the structural layout of `t3-code`:
- left collapsible sidebar
- main chat thread
- visible diffs
- settings surface

But reinterpret it through the current desktop theme rather than copying it visually.

## Product constraints
- This is a separate app repo, not part of `niri-quickshell-bar`
- v1 should be real chat + real live observation, not observer-only
- The bar can later act as launcher/status entry point for this app
